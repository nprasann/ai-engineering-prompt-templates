# Release Checklist

## Code
- [ ] Diffs reviewed.
- [ ] Tests pass.
- [ ] Linting/formatting complete.
- [ ] No unrelated changes.

## Security and Privacy
- [ ] No secrets, tokens, credentials, or private keys.
- [ ] No production data or sensitive logs.
- [ ] Example values are generic.
- [ ] Dependencies and licenses reviewed.

## Documentation
- [ ] README updated.
- [ ] Setup and test commands verified.
- [ ] Known limitations documented.

## Publishing
- [ ] Version or release notes prepared.
- [ ] Public repository settings reviewed.
- [ ] Final human review completed.
